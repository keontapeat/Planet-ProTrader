We’ve spent enough time chasing the native Linux wheel.  
Your VPS already has Wine + the real Windows MT5 terminal running, so the
quickest, 100 % guaranteed route is:

1. Use the **Python that’s inside Wine** (it can always install MetaTrader5).
2. Launch the bot with that Wine Python (inside the existing Xvfb :1).

Copy-paste the block below EXACTLY as shown after you SSH into the VPS.
That’s it—no edits, no hunting for wheels.

#─────────────────────  BEGIN COPY BLOCK  ─────────────────────
# 0) Make sure you are on the VPS and Xvfb :1 + terminal64.exe are running.

# 1) Define the Wine prefix (adjust if you installed Wine elsewhere)
export WINEPREFIX=$HOME/.wine

# 2) Create ONE shared venv inside Wine (do this only once)
wine "$WINEPREFIX/drive_c/Program Files/Python39/python.exe" -m venv C:\\opt\\goldex-bot\\winevenv

# 3) Install MetaTrader5 + helper libs with Wine’s pip
wine C:\\opt\\goldex-bot\\winevenv\\Scripts\\pip.exe install MetaTrader5 pandas python-dotenv

# 4) Quick connectivity test (uses creds in /opt/goldex-bot/.env)
DISPLAY=:1 \
wine C:\\opt\\goldex-bot\\winevenv\\Scripts\\python.exe - <<'PY'
import MetaTrader5 as mt5, os, sys
from dotenv import load_dotenv
load_dotenv('/opt/goldex-bot/.env')
login    = int(os.getenv('DEMO_LOGIN', 0))
password = os.getenv('DEMO_PASSWORD', '')
server   = os.getenv('DEMO_SERVER', '')
if not mt5.initialize(login=login, password=password, server=server):
    print("❌  MT5 init failed:", mt5.last_error()); sys.exit(1)
print("✅  Connected — Balance:", mt5.account_info().balance)
mt5.shutdown()
PY

# 5) If the ✅ line appears, run the real bot in tmux so it stays alive:
tmux new -s goldex -d "
  export DISPLAY=:1 &&
  wine C:\\opt\\goldex-bot\\winevenv\\Scripts\\python.exe C:\\opt\\goldex-bot\\goldex_bot.py
"
echo '🚀  GoldexBot launched inside tmux session "goldex".'
#──────────────────────   END COPY BLOCK   ────────────────────
