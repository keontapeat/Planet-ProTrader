================================================================================
“No such module ‘Supabase’” – 6 THINGS TO CHECK
================================================================================
You said the package is installed, yet the compiler can’t see it.  
99 % of the time it’s one of the issues below.

────────────────────────────────────────────────────────────────
1.  Wrong PRODUCT linked to the target
────────────────────────────────────────────────────────────────
• In **Project navigator** → select *GOLDEX AI* target  
• Build Phases ▸ **Frameworks, Libraries & Embedded Content**  
  ☐  Do you see “Supabase.framework”?  
  If not:  
    • Build Settings ▸ Swift Package Dependencies → + → Supabase → Add.

────────────────────────────────────────────────────────────────
2.  Package not resolved / stuck in cache
────────────────────────────────────────────────────────────────
• File ▸ Packages ▸ *Reset Package Caches*  
• File ▸ Packages ▸ *Resolve Package Versions*  
• Clean build folder (Shift-Cmd-K) → Re-build.

────────────────────────────────────────────────────────────────
3.  Supabase-swift requires Xcode 15.0+
────────────────────────────────────────────────────────────────
Open Xcode ▸ About.  
If you’re on 14.x, upgrade—package uses Observation macros unavailable earlier.

────────────────────────────────────────────────────────────────
4.  Duplicate “Supabase” module shadowed / excluded
────────────────────────────────────────────────────────────────
If you manually dragged the repo into the project earlier, remove it:
• In navigator delete any group named “Supabase” (Move to Trash).  
Then *Swift Package Manager* takes over.

────────────────────────────────────────────────────────────────
5.  Target platform mis-match
────────────────────────────────────────────────────────────────
Your iOS target must be iOS 13+ (the package’s minimum).  
Project ▸ General ▸ Deployment Info → iOS 17 / 16 / 15 etc.

────────────────────────────────────────────────────────────────
6.  Derived Data corruption
────────────────────────────────────────────────────────────────
• Xcode ▸ Settings ▸ Locations ▸ click arrow next to Derived Data  
• Trash the GOLDEX_AI folder. Re-build.
