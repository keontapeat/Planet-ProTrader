“No such module ‘Supabase’” – HOW TO RESOLVE
================================================================================
Reason
───────
Xcode cannot find the Supabase Swift package.  
Either the package was never added, or it isn’t linked to the **GOLDEX AI**
target.

Steps to fix
────────────
1️⃣  Add / Verify the Package  
    1. Xcode ▸ Settings… ▸ **Packages** tab.  
    2. Click “+” → enter:
         https://github.com/supabase-community/supabase-swift.git  
    3. Choose *Up to Next Major*.  
    4. In the *Add to Target* list, tick **GOLDEX AI**.  
    5. Finish → Xcode resolves the package.

2️⃣  Clean & Build  
    • Shift-Cmd-K (Clean Build Folder)  
    • Cmd-B (Build)  
    The error should disappear.

3️⃣  If You Use `SupabaseConfig` (recommended)  
    • Delete **SupabaseManager.swift** (duplicate).  
      Cmd-Delete → Move to Trash.  
    • All code uses:
          let client = SupabaseConfig.shared

4️⃣  If You KEEP `SupabaseManager.swift`  
    • Fix the bad URL:
          URL(string: "https://tetuqchhlrhpbwatrods.supabase.co")!
      (remove the double “https://”)  
    • Ensure the file is part of the **GOLDEX AI** target
      (File ▸ Inspect Sidebar ▸ Target Membership ☑).

5️⃣  Verify
    • Build succeeds.  
    • Run app → no crash on Supabase import.  
    • Console shows no fatal errors from SupabaseConfig.

That’s it—once the package is properly linked, `import Supabase` compiles. 🚀
