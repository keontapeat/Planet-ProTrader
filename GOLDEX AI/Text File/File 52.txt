WHY “docker compose” FAILED & 90-SECOND FIX
──────────────────────────────────────────────────────────────────────────────
1. Ubuntu’s `docker.io` package ships **Docker Engine** only.  
   The Compose V2 plugin (command `docker compose …`) is a **separate** pkg
   called `docker-compose-plugin`.

2. Let’s install that plugin, then start the stack.

RUN THE $ COMMANDS BELOW
────────────────────────────────────────────
$ apt update -y
$ apt install -y docker-compose-plugin      # adds /usr/libexec/docker/cli-plugins/docker-compose

# verify
$ docker compose version                    # should print “Docker Compose version v2.x.x”

# copy .env correctly (you missed the destination path)
$ cp .env /opt/goldex-bot/.env

# copy bot if not in /bot
$ cp goldex_bot.py /opt/goldex-bot/bot/

# START the container
$ cd /opt/goldex-bot
$ docker compose up -d                      # pulls image & starts bot

# Tail logs
$ docker compose logs -f                    # look for ✅ Connected — Balance

That’s it—installing `docker-compose-plugin` resolves the “unknown command”
error.  Two minutes and you’ll see your bot live. 🚀
