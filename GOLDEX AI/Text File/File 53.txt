ROOT CAUSE
──────────────────────────────────────────────────────────────────────────────
• You installed Docker from Ubuntu’s repo (`docker.io`) ONLY.  
  That gives you `docker` but NOT the new **Compose V2** sub-command  
  (`docker compose …`).  
• `docker-compose-plugin` lives in Docker’s **official** repo, not Ubuntu’s.

90-SECOND FIX
─────────────
Run the 7 lines below.  They add Docker’s repo, then install the plugin.

$ apt remove -y docker-compose         # clean old v1 if present (ignore errors)
$ mkdir -p /etc/apt/keyrings
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
      gpg --dearmor -o /etc/apt/keyrings/docker.gpg
$ echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
  > /etc/apt/sources.list.d/docker.list
$ apt update -y
$ apt install -y docker-compose-plugin
$ docker compose version      # should print “Docker Compose version v2.x.x”

START THE BOT (same as before)
──────────────────────────────
$ cd /opt/goldex-bot
$ docker compose up -d        # pulls image & starts container
$ docker compose logs -f      # watch for ✅ Connected — Balance

TROUBLESHOOTING SHORTCUTS
─────────────────────────
• Still says unknown command? → `which docker-compose` should show  
  `/usr/libexec/docker/cli-plugins/docker-compose`. If not, plugin not in $PATH.  
• VPS low on disk?  `df -h` (need ~3 GB free for image + logs).  
• Image pull slow?  Just rerun `docker compose up -d`—it resumes.

You install the plugin once; after that, `docker compose` works forever. 
Run the seven lines, then up / logs, and you’re done. 💥
